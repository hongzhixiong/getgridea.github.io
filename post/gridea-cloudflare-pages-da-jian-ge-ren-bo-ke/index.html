Gridea Pro + Cloudflare Pages 搭建个人博客

默认分类 · 2026-05-19 · 5 分钟阅读
技术折腾

Gridea Pro（配合Cloudflare Pages）
特点：静态博客写作客户端，免费，有活跃用户制作主题。
费用：软件免费；Cloudflare Pages托管免费。
域名绑定：Cloudflare Pages支持自定义域名绑定。
管理后台：Gridea提供本地客户端管理，写作后一键推送到Cloudflare Pages；虽然需要简单部署，但无需自己搭建服务器，相当于半托管方案。
适用场景：适合技术基础较弱但愿意接受极简部署流程的用户。
对比Bitcron：虽然Bitcron更傻瓜，但Gridea和Bitcron一样，是成本较低的解决方案。
对比Hugo：部署简单，维护简单，适合零基础和不想折腾的用户。

一、准备工作
与旧版相同，你需要注册两个免费账号：

GitHub账号 - 用于存放博客文件
Cloudflare账号 - 用于托管和加速博客

二、获取并安装 Gridea Pro

  1. 下载安装包
    访问 Gridea 官方仓库获取最新版本。Gridea Pro 是针对不同架构优化的版本，根据你的系统选择合适的安装包：

Windows 64位系统：选择 Gridea.Pro_v1.2.0_windows_amd64_setup.exe（安装版）或 Gridea.Pro_v1.2.0_windows_amd64_portable.exe（便携版）
Windows ARM设备（如 Surface Pro X）：选择 Gridea.Pro_v1.2.0_windows_arm64_portable.exe
MacOS用户：选择对应的 .dmg 文件

  1. 安装与最小初始配置
    Gridea 的核心理念是最小初始配置——你不需要填满所有设置才能开始写作。建议先快速完成以下基础设置：

源文件夹设置：建议将源文件夹设置在云盘目录（如 OneDrive、百度网盘），便于多设备同步
博客基本信息：填写博客名称、作者、描述等

三、GitHub 端配置

  1. 创建仓库
    登录 GitHub 后，创建一个公开仓库，仓库名必须为：你的用户名.github.io
  2. 生成访问令牌（Token）

进入 GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)
点击 Generate new token，权限勾选 repo（全部勾上）
生成后立即复制并保存好Token

四、Gridea Pro 远程配置

  1. 配置远程连接
    在 Gridea Pro 的 设置 → 远程 中填写：

平台：选择 GitHub
仓库名：你的用户名.github.io
分支：master
用户名：你的 GitHub 用户名
令牌：粘贴刚才保存的Token
域名/ CNAME：如果你有自定义域名，填写你的域名

  1. 检测远程连接
    点击 检测远程连接。如果失败，**开启网络代理（科学上网）**或使用 Proxifier 软件代理 Gridea 的同步操作。
  2. 首次同步
    检测成功 → 点击 保存 → 点击左下角 同步 按钮。稍等片刻，访问 https://你的用户名.github.io 应能看到默认博客。

五、Cloudflare Pages 部署

  1. 将域名托管到 Cloudflare
    第一步：在 Cloudflare 添加站点

登录 Cloudflare Dashboard，点击 添加站点
输入你的域名，选择 免费计划
复制 Cloudflare 分配的两个名称服务器地址

第二步：在 Spaceship 修改名称服务器

登录 Spaceship → 域管理器 → 找到你的域名
进入 高级 DNS → 名称服务器，选择 自定义名称服务器
删除 Spaceship 默认的名称服务器，粘贴 Cloudflare 的两个地址
点击 保存 DNS 设置

第三步：验证托管状态

回到 Cloudflare 点击 立即检查名称服务器
状态从 待处理 变为 活动 后即完成（约10分钟至48小时）

  1. 创建 Pages 项目

在 Cloudflare 左侧菜单选择 Workers & Pages
点击 创建应用程序 → 选择 Pages 标签
点击 连接到 Git，授权并选择 用户名.github.io 仓库

  1. 配置构建设置

项目名称：随便填（如 my-blog）
生产分支：master
构建设置：框架预设选 None，构建命令留空，输出目录留空
点击 保存并部署

  1. 绑定自定义域名

在 Pages 项目 → 设置 → 自定义域 → 添加自定义域
输入你的完整域名（如 blog.yourdomain.com）
Cloudflare 会自动添加 DNS 记录并签发 SSL 证书

六、创作与发布流程

  1. 写文章

    Gridea Pro 支持Monaco Markdown编辑器，提供 VS Code 级别的编辑体验。创建文章时：

填写文章标题
使用 Markdown 语法编辑内容
添加标签进行分类（建议使用 3-5 个核心标签，避免过度分类）
设置封面图（支持外链图片，可提升页面加载速度）
自定义文章 URL

  1. 主题选择与定制
    Gridea Pro 提供多种内置主题（如 fly、notes、paper、simple）。在 设置 → 主题 中：

预览并选择喜欢的主题
修改主题配置（如皮肤颜色、内容区最大宽度、正文文字大小等）
如果要深度定制，可以编辑主题的样式文件（如 colors.less）或模板文件

  1. 开启评论系统（可选）
    Gridea 内置支持 Gitalk 和 DisqusJS 评论系统。配置步骤：

在 GitHub Settings → Developer settings → OAuth Apps 注册应用
点击 New OAuth App，填写应用信息：
Application name：任意，如 Gridea-Gitalk
Homepage URL：你的博客首页地址（如 https://xxxx.com）
Authorization callback URL：与 Homepage URL 保持一致（注意：如果你绑定了自定义域名，这里不要带 www，否则可能导致点击登录后跳转到首页的问题）
创建成功后，复制 Client ID 和 Client Secret 保存到本地
将获取的 Client ID 和 Client Secret 填入 Gridea 的评论设置中

  1. 同步发布

写作完成后点击 预览 查看效果
确认无误后点击 同步，Gridea 会自动推送到 GitHub
Cloudflare Pages 检测到更新后会自动重新部署，无需手动操作

七、多设备协同方案
利用 Gridea Pro 的自定义源文件夹功能，可以实现多设备写作同步：

将源文件夹设置在云盘目录（如 OneDrive）
在不同设备上安装 Gridea Pro 并指向同一目录
启用自动保存功能
写作内容会自动同步到所有设备

总结

这套 Gridea Pro + Cloudflare Pages 方案依然是目前成本最低（几乎免费）且最接近Bitcron“写完即发布”体验的选择。
