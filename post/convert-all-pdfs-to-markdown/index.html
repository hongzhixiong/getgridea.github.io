将知识库中的pdf都转换成markdown

2026-06-03 学习相关

按目前的说法，Markdown（.md）格式是AI工具分析理解的最佳选择，其次是结构化HTML，而已经OCR的PDF虽然可用但存在明显局限。

我自己的阅读习惯是下载扫描版的pdf，再用ABBYY制作双层pdf，这样阅读起来既美观又支持搜索和复制。接下来的问题就是如何将双层pdf转换成markdown？其实现在有很多工具，比如MinerU以及专注扫描版pdf转换的Pdf-Craft，但是由于我手上都是包含文字层的双层PDF，这些一站式解决工具往往会对双层PDF重新进行OCR，这会导致：

  1. 重复劳动：明明已经有准确文字层，却要再次OCR，浪费时间和算力
  2. 可能引入新错误：二次OCR可能产生新的识别错误，反而降低原有质量
  3. 效率下降：重新OCR需要额外处理时间，尤其对大批量文档

更何况像MinerU这种工具的本地客户端和线上网页本质上是云端解析，文件会被传输到远程服务器处理。所以页数上会有限制，如单文件页数少于600页。如果将pdf进行分割分别处理的话，又会有后续markdown文件的合并问题，很麻烦。

所以我目前的流程是：扫描版PDF → ABBYY OCR → Word文档 → Pandoc → Markdown。
Pandoc是目前最成熟、兼容性最强的开源文档转换引擎，原生支持.docx到.md的语义化转换，能准确识别标题级别、段落样式、表格结构及内嵌图片，并支持批量处理与路径控制。
Pandoc如何安装？这里介绍包管理器安装法，即通过命令自动安装，好处是安装后不用配置环境。

一、安装Windows 官方包管理器 — Winget

  1. 按 Win + S 键打开搜索框，输入“Microsoft Store”并打开应用商店
  2. 在商店搜索栏中输入“App Installer(应用安装程序)”，如若搜索不到，请直接访问这个地址：https://apps.microsoft.com/detail/9NBLGGH4NNS1
  3. 点击“获取”或“安装”按钮，等待自动下载和安装完成
  4. 安装完成后，打开CMD或PowerShell，输入 winget --version 验证安装成功

二、使用Winget安装pandoc

在刚才的命令行窗口中输入以下代码：

winget install --source winget --exact --id JohnMacFarlane.Pandoc

安装完成后在命令行窗口输入以下命令验证是否成功：

pandoc --version

三、使用Pandoc将Word文件转换为Markdown

  1. 在当前文件夹中使用CMD命令

    打开文件资源管理器，导航到目标文件夹。点击文件资源管理器的地址栏，使其处于可编辑状态。直接输入 cmd 并按回车键，这样会立即打开以当前目录为工作路径的命令提示符窗口。这种方式无需任何额外设置，操作即时生效。

  2. 单个Word文件的转换命令

    进入Word文件所在的目录，在命令行（CMD或PowerShell）中执行以下命令：

pandoc "你的文档.docx" -o "输出文档.md"

这个命令会将指定的Word文档转换为Markdown格式，并保留标题、列表、代码块等基础格式。如果文件名或路径包含空格，记得使用英文双引号包裹起来。

3.批量转换整个文件夹中的Word文件

进入Word文件所在的文件夹后，运行以下命令：

for %f in (*.docx) do pandoc "%f" -o "%~nf.md"

这条命令会遍历当前文件夹中所有.docx文件，并逐个转换成同名的Markdown文件。
