再也不折腾DNS了

2026-07-08 · 5 分钟阅读
技术折腾

自用Nikki dns模块设置
dns:
enable: true
cache-algorithm: arc
listen: ‘[::]:1053’
ipv6: false
respect-rules: true
enhanced-mode: fake-ip
fake-ip-range: 198.18.0.1/16
fake-ip-filter-mode: blacklist
fake-ip-filter:
- “rule-set:fakeipfilter_domain”
default-nameserver:
- 223.5.5.5
- 119.29.29.29
proxy-server-nameserver:
- https://223.5.5.5/dns-query
- https://1.12.12.12/dns-query
direct-nameserver:
- https://223.5.5.5/dns-query
- https://1.12.12.12/dns-query
nameserver:
- https://8.8.8.8/dns-query#RULES
- https://1.1.1.1/dns-query#RULES
prefer-h3: true
direct-nameserver-follow-policy: false
nameserver-policy:
‘rule-set:fakeipfilter_domain’:
- https://223.5.5.5/dns-query
- https://1.12.12.12/dns-query
‘geosite:cn’:
- https://223.5.5.5/dns-query
- https://1.12.12.12/dns-query

dns:
enable: true

【缓存算法】使用 ARC（自适应替换缓存），相比传统的 LRU 算法，它能更好地平衡热点域名和近期域名的缓存，命中率高，减少重复查询。

cache-algorithm: arc

【监听端口】监听所有 IPv6 和 IPv4 地址的 1053 端口，作为本地的 DNS 服务端。

listen: ‘[::]:1053’

【IPv6 关闭】不返回 AAAA (IPv6) 记录。因为很多网络环境 IPv6 支持不好，关闭可避免应用走到 IPv6 导致连接超时。

ipv6: false

【核心设置：尊重规则】DNS 查询结果的处理将严格遵循路由规则。配合下面 nameserver 里的 #RULES 标签使用，确保 DNS 请求本身的走向（走代理还是直连）由规则决定，防止 DNS 泄漏。

respect-rules: true

【核心设置：Fake-IP 模式】启用 fake-ip 增强模式。访问代理域名时，Clash 直接返回一个假 IP（198.18.x.x），应用拿到假 IP 立即发起连接，Clash 在代理握手时才把真实域名发给代理服务器，极大降低首次连接延迟。

enhanced-mode: fake-ip

【Fake-IP 池】分配假 IP 的范围，198.18.0.0/16 是保留地址段，不会与公网 IP 冲突。

fake-ip-range: 198.18.0.1/16

【Fake-IP 过滤模式】黑名单模式（blacklist）：意味着“只有匹配到下面过滤列表的域名才返回真实 IP，其余统统返回假 IP”。

fake-ip-filter-mode: blacklist

【Fake-IP 过滤列表】局域网设备、需要真实 IP 才能登录的站点（如银行、游戏）等，不分配假 IP，去请求真实 IP。

fake-ip-filter:
- “rule-set:fakeipfilter_domain”

【底层引导 DNS】仅用于解析 DoH/DoT 域名（例如解析 dns.alidns.com 的 IP）。必须是纯 IP 的传统 DNS，否则会陷入死循环。这里用了阿里和腾讯的国内公共 DNS。

default-nameserver:
- 223.5.5.5
- 119.29.29.29

【代理节点域名 DNS】专门用于解析你订阅的代理服务器节点的域名（如 node.proxy.com）。必须在直连网络下解析，否则代理连不上，形成死锁。

proxy-server-nameserver:
- https://223.5.5.5/dns-query
- https://1.12.12.12/dns-query

【直连域名 DNS】匹配到直连规则的国内域名，使用这些 DNS 解析，确保拿到距离最近的国内 CDN 节点 IP，网速最快。

direct-nameserver:
- https://223.5.5.5/dns-query
- https://1.12.12.12/dns-query

【主 DNS】用于解析代理域名（国外域名）。后面的 #RULES 是 Clash Meta 特有标签，意思是“这个 DNS 请求本身怎么发出去，交给路由规则决定”。

配合上面的 respect-rules: true，解析国外域名时的 DNS 请求会自动走代理发出去，避免本地运营商 DNS 污染，且防止 DNS 泄漏。

nameserver:
- https://8.8.8.8/dns-query#RULES
- https://1.1.1.1/dns-query#RULES

【HTTP/3 优先】优先使用 HTTP/3 (QUIC) 协议进行 DoH 查询，基于 UDP，握手更快，降低 DNS 查询延迟。

prefer-h3: true

【直连 DNS 不跟随策略】直连 DNS 独立工作，不再受 nameserver-policy 的二次干扰，避免逻辑混乱和查询变慢。

direct-nameserver-follow-policy: false

【分流前置拦截器】它提前把“明确知道是国内”的域名（geosite:cn）和“绝对不能用假IP/国外DNS”的域名（fakeipfilter_domain）摘出来，直接送去国内 DNS 解析真实 IP，优先级最高。

nameserver-policy:
# 假 IP 过滤列表里的域名（如局域网、银行等），强制用国内 DoH 解析真实 IP
‘rule-set:fakeipfilter_domain’:
- https://223.5.5.5/dns-query
- https://1.12.12.12/dns-query
# 属于国内站点但未被 fake-ip 过滤的域名，也强制用国内 DoH 解析，确保拿到真实国内 IP（而不是假 IP），有利于国内直连的 CDN 优化
‘geosite:cn’:
- https://223.5.5.5/dns-query
- https://1.12.12.12/dns-query
